# Proposal: 供应商绩效分析系统

## 1. 问题陈述

当前供应商绩效考核流程依赖手动 Excel 表格处理，存在以下痛点：
- 多部门协同打分散乱，缺乏统一管理平台
- 数据统计分析依赖人工计算，效率低且易出错
- 缺乏可视化展示，决策依据不足
- 历史数据难以追溯，趋势分析困难

## 2. 目标

构建一个基于 Spring Boot + React + MySQL 的供应商绩效分析系统，实现：
- 自动化的数据导入与处理
- 多维度的绩效分析与可视化
- 支持各部门独立打分与数据汇总
- 完整的权限管理体系
- 数据持久化与历史追溯

## 3. 成功标准

| 指标 | 目标值 |
|------|--------|
| 页面加载时间 | ≤ 3秒 |
| API 响应时间 | ≤ 1秒 |
| Excel 导入速度 | ≤ 10秒/100条记录 |
| 系统可用性 | ≥ 99.5% |
| 支持并发用户 | ≥ 50人 |

## 4. 变更范围

### 4.1 新增模块
- 数据上传模块：Excel 文件上传、解析、验证
- 总览趋势模块：统计卡片、趋势图表、等级分布
- 维度对比模块：雷达图、热力图对比分析
- 排名变动模块：月份对比、排名变化分析
- 明细数据模块：完整考核明细展示与导出
- 供应商池模块：供应商信息管理
- 手动打分模块：多部门协同打分流程
- 系统管理模块：用户、角色、权限、菜单管理

### 4.2 技术架构
- 后端：Spring Boot 3.2 + EasyExcel + JWT
- 前端：React 18 + Ant Design 5 + ECharts
- 数据库：MySQL 8.0，表名前缀 `sp_`

## 5. 依赖与前置条件

- JDK 21+
- Maven 3.9+
- Node.js 20+
- MySQL 8.0+

## 6. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Excel 格式不统一 | 解析失败 | 提供标准模板，严格验证 |
| 多部门数据冲突 | 数据不一致 | 唯一索引约束，合并规则明确 |
| 权限控制复杂 | 安全漏洞 | 基于角色的细粒度权限管理 |
| 大数据量性能 | 查询缓慢 | 索引优化，分页查询 |

## 7. 验收标准

- 所有 API 接口正常响应
- 前端页面功能完整可用
- Excel 导入/导出功能正常
- 权限控制生效
- 数据持久化正确
